AWS Service types

發表於2022-02-22|更新於2022-02-22|雲端知識
|閱讀量:

Computing Resources

Amazon EC2

  • AMIs
  • Custom AMI
  • Instance Type
  • User Data
  • Tag
  • Lifecycle (Pending, Running, Stop, Reboot, Terminate)

    AWS Lambda

  • Serverless function

Connection

AWS VPC

  • VPN Flow:
    IGW <> Route Table<>NACL<>Subnet<>Security group<>Instance
  • Across Multiple AZs
  • Subnet default: 前四後一
  • Network ACL: Deny Traffic
  • Route table
  • NAT Gateways: 只支援裡面到外面
  • IGW (0.0.0.0/0)
  • Elastic IP Address
  • ENI: Elastic Network Interface (Same AZ, Accross EC2)

    AWS DX

  • 需要向 AWS 申請
  • Dirrect Connect

Serverless

Lambda Function

  • Lambda Layer (At most 5)
  • Source Code
  • Configurations (Runtime max, RAM max)
  • Stateless
  • Fully managed
  • handle_name
  • Version control
  • All run in VPC
  • Execution Permission

    Amazon API Gateway

  • Allow to create API
  • Running on EC2, Lambda, HTTP(s) base application
  • DDoS protection
  • Authenticate & Authorize requests
  • Endpoint: Edge-optimized, Regionsl, Private

    AWS step funciton

  • Allow to Retry
  • Flow controller between Lambda
  • Self programming language
  • Low-code visual workflow to manage serverless architecture
  • Workflow: Parallel, Series, if/else, …
  • State machine

    AWS X-ray

  • Monitor of Serverless tool, 甘特圖
  • 顯示不同 serverless 啟用時間

Container

Amazon ECS

  • Docker container
  • Linux, Window

    Amazon Fargate

  • Serverless Container
  • Linux

Storage

Amazon DMS

  • Database migration
  • Use case: Migrate, Replicate

    Amazon EBS

  • Bulk storage: HDD, SDD
  • Instance Store
  • Can only up scale
  • IOPS, Throughput
  • EBS Optimized
  • Snapshot: Across Region, encrypted, Version control

    Amazon S3

  • Fully managed
  • Use case: Static Website, Endpoints for Download, Backup tool…
  • CAP Thm = Brewer’s thm (只能三選二)
    1. Consistency: Strong read-after-write consistency model
    1. Availability: Eventual Consistency
  • Bucket: Regional service
  • Object Storage: Object URL(bucket name, object key)
  • Object key are partitioned by “prefix”
  • Storage Classes:
    1. S3 Standard
    1. S3 Intelligent-Tiering
    1. S3 Standard IA
    1. S3 One Zone IA
    1. S3 Glacier

      Amazon EFS & FSx

  • Shared File System
  • EFS:
    1. Linux-based
    1. Mount Target (Cross AZs)
    1. Fully managed
  • FSx:
    1. window-based

      Amazon RDS

  • Relational Database Service
  • 垂直擴展
  • Fully managed
  • Subnet group(Multi-AZ): Primary DB, Secondary DB
  • Read Replica
  • Automated backups & Manually snapshot
  • Backup window
  • Encryption: AWS KMS
  • Maintenance windows
  • Auto Scaling

    Amazon Dynamo DB

  • Key-Value Database
  • Primary key = Partition Key + Sort Key
  • Fully managed
  • Use case: 高 I/O 工作負載
  • Strong consistency: 2 RCU
  • Eventual consistency: 1 RCU
  • Dynamo DB Stream
  • Global Table
  • Auto Scaling: RCU or WCU

Secure

Security Group

  • “Allow” resource to be access
  • Virtual firewall for instances
  • Stateful
  • Support, Allow rules only

    AWS IAM

  • Root User > IAM User, Admin Group > IAM Group
  • Who: Root User, IAM User, IAM Group
  • What, To Whom (JSON): IAM Policy, IAM Role

    AWS WAF

  • OSI 7th: Application & API
  • CloudFront, API Gateway, ALB

    AWS Shield

  • OSI 3th
  • DDoS Protection
  • Route53, CloudFront, ELB

    Artifact

  • 紀錄現在 AWS 合規有哪些
  • SOC, PCI, BAA, NDA

    AWS Trusted Advisor

  • 最佳實踐的提醒
  • 針對整個 Account
  • Real time guidance
  • Security check
  • Inspect: Cost, Performance, Security, FT, Service Limit

Monitor

Amazon CloudWatch

  • Resource Aspect
  • Complete visibility of cloud resources and service (資源裡的服務)
  • Metrics, Alarm, Rule & Event, Schedule, Logs

Amazon Config

  • Account Aspect
  • Assess, audit, and evalutate the configurations of your AWS resources (帳號資源)
  • “Config Rules” 來管控 Account 的 Policy (強制)

    Amazon CloudTrail

  • Account Aspect
  • Account API calling Record (登入登出)

Amazon Inspector

  • Automation Inspect
  • 偵測應用程式方面的弱點
  • 依據國際組織的安全標準幫忙健檢

    Amazon Macie

  • 自動化偵測
  • Only S3 bucket
  • Only Advice

Amazon SNS

  • pub/sub msg
  • publisher, topic, subscriper

    Amazon SQS

  • For Queueing Request (Buffer)
  • Send/Receive msg
  • Fully managed msg
  • Asynchronous
  • Queue Types: Standard, FIFO
  • Msg lifecycle
  • API Call: Short/Long polling, Dead Letter queue
  • Use case: Work queues, Buffer operations, request reloading, trigger scaling

    Amazon ELB

  • Only Between AZs
  • 可以 external (Public) 也可以 internal (Private) 導流服務
  • Target: Computing unit (EC2, Containers, IP address)
  • Types:
    1. Application Load Balancer (ALB): 7th layer
    1. Network Load Balancer (NLB): 4th layer
    1. Classic Load Balancer: Previous generation
  • Target Group: 可跨 AZs

    Amazon Route 53

  • Global Service (Across Regional)
  • Target: DNS load balancing (導流服務)
  • Domain name: monthly subscribe
  • Routing Policy:
    1. simple routing
    1. Failover routing
    1. Geolocation routing: Request IP & Endpoint
    1. Geoproximity routing: Server loaction (Traffic flow only)
    1. Latency routing
    1. Multi-value answer routing
    1. Weighted routing

Auto Scaling

RTO

  • How long system needs for recovering from a disaster

    RPO

  • A measurement of the maxium tolerable amount of data to lose

    AWS Auto Scaling

  • Horizontal Scaling & Vertical Scaling
  • Through CloudWatch

Machine Learning

Amazon SageMaker & DeepRacer

Amazon Lex

  • Call Center

    Amazon Polly & Transcribe

  • 語音文字轉換

    Amazon Rekognition

  • 影像辨識服務

IoT

Amazon IoT Core

Amazon IoT Greengrass

Amazon IoT Analytics

AR & VR

Amazon Sumerian

Edge Location

  • 與當地企業合作 (不一定自己機房)

    Amazon CloudFront

  • stores subset of data
  • CDN service
  • Global Edge Location, Regional Edge Location
  • Caching static Content
  • TTL
  • Origin Access Identity (OAI)

    Amazon Route 53

    AWS Lambda@Edge

    Amazon ElastiCaching

  • Database Caching
  • Finding heavy-hitting, high usage or high volume requests inundates database
  • Memcached: 需清楚定義
  • Redis: 針對 Key 做索取

Local Zone

  • Extension of AWS Region
  • 與 Edge Location 差別: AWS 自己的機房, 支援服務種類更多
  • 提升 Single-digit latency
  • Support: EC2, EBS, FSx, ELB, VPC, EMR, RDS

Automation Deployment

Amazon CloudFormation

  • 類似 Azure ARM template 功能
  • Workflow: Template > 放進 CloudFormation > CloudFormation 複製

    Amazon Elastic Beanstalk

  • 懶人部屬, 最簡單
  • 代工不用付費, 只需付費用到的功能

Payment

Pricing Calculator

  • Counting “預算”

    AWS Budget

  • Billing Alert, Budgets reports
  • Centralized management

    AWS IAM Checklist

  • 建議提升安全性的方法
  • Advice
  • MFA, Delete Root Access Keys, IAM passwork policy

    Pricing Model

  • On-demand: pay for what you want
  • Spot Instance: Cheaper than on-demand, but 隨時會被收回
  • Reserved Instance: “年”為單位, 簽的單位為物理資源(EC2, S3…)
  • Saving Plan: 簽的單位為 IT 資源(CPU, Memory…)
  • Dedicated Instance: pyhsically isolated
  • Dedicated host: 租下整個機櫃 (Software licenses)
